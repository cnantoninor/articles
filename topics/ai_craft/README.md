# AI Craft: Development Practice, Design, and Identity

## Overview

**Status:** Migrated (TAM3 + TAM4 + TAM5)
**Created:** 2026-02-22
**Last Updated:** 2026-02-22

Articles on how AI changes software development practice: the psychology of deleting AI-generated code, moving from vibe coding to vibe designing, and how identity and memory in systems connect to Domain-Driven Design (e.g., Ship of Theseus).

## Key Questions

1. What does the psychology of deleting AI-generated code reveal about our changing relationship with software development?
2. How can we move beyond "vibe coding" to use AI for deeper problem modeling and design thinking?
3. How do we model identity, memory, and context in software (Entity/Value Object, Event Sourcing, Bounded Contexts)?

## Central Concepts

- **Intent ownership, error traceability, knowledge transferability**: Three tests for AI's impact on development
- **Vibe Designing**: Pre-prompting domain modeling, Socratic questioning, assumption tracking, design exploration
- **Ship of Theseus**: Identity as narrative; Entity vs Value Object; Event Sourcing; Bounded Contexts
