# Domain Terminology

## Reference
See `GLOSSARY.md` for shared definitions across all topics.

## Key Concepts (Epistemic Debt Topic)
- **Epistemic debt**: Code that works but nobody understands
- **Epistemic warrant**: Justification for claiming to understand something
- **Solutioning trap**: Jumping to implementation before defining the problem
- **Construction vs. Curation paradigm**: Traditional authorship vs. probabilistic selection
- **Epistemic boundary**: Points where meaning translates between forms
- **Circular validation**: LLM-generated tests validating LLM-generated code

## Terms to Avoid or Clarify
- "AI" alone — specify type (LLM, ML model, etc.) when relevant
- "Understand" for AI — use "process" or "generate" instead
- "Know" for developers — clarify what kind of knowledge (procedural, declarative, etc.)
- Unqualified absolutes ("always", "never") — hedge appropriately
