# Codebase Structure

**Analysis Date:** 2026-01-26

## Directory Layout

```
ai-articles/
├── .cursor/                # Cursor IDE plans and metadata
│   └── plans/              # Generated planning artifacts
├── .planning/              # Internal planning/mapping artifacts
│   └── codebase/           # Codebase documentation
├── .cursorrules            # AI assistant style guide and conventions
├── CLAUDE.md               # Symlink to .cursorrules
├── README.md               # Repository documentation
├── GLOSSARY.md             # Shared terminology definitions
├── scripts/                # Export automation scripts
│   ├── export-docx.sh      # Markdown → DOCX conversion
│   ├── export-slides.sh    # Marp → PPTX conversion
│   ├── export-pdf.sh       # Markdown/Marp → PDF conversion
│   └── export-all.sh       # Batch export orchestrator
├── templates/              # Content templates
│   ├── article.md          # Article template with YAML front-matter
│   ├── slides.md           # Marp presentation template
│   └── research.md         # Research notes/README template
└── <topic-name>/           # Topic directory (e.g., epistemic_debt/)
    ├── README.md           # Topic overview, status, gaps
    ├── article.md          # Main article draft
    ├── slides.md           # Presentation slides (Marp format)
    ├── raw_material/       # Working notes, outlines, conversation logs
    ├── references/         # Literature reviews, citations
    ├── assets/             # Images, diagrams, media files
    ├── artifacts/          # Alternative artifact storage
    └── exports/            # Generated DOCX, PPTX, PDF files
```

## Directory Purposes

**.cursor/**
- Purpose: Stores Cursor IDE-specific metadata and planning artifacts
- Contains: AI-generated plans, session data
- Key files: `/plans/ai-friendly_article_repo_dc9c4233.plan.md`

**.planning/**
- Purpose: Internal documentation about the codebase itself
- Contains: Codebase mapping documents (this file, ARCHITECTURE.md, etc.)
- Key files: `/codebase/ARCHITECTURE.md`, `/codebase/STRUCTURE.md`

**Root Directory:**
- Purpose: Repository-level documentation and configuration
- Contains: README, style guide, glossary, configuration files
- Key files:
  - `README.md`: Getting started, quickstart, structure overview
  - `.cursorrules`: Complete AI authoring guidelines (4564 bytes)
  - `CLAUDE.md`: Symlink to `.cursorrules` for visibility
  - `GLOSSARY.md`: Domain terminology and definitions

**scripts/**
- Purpose: Export automation workflows
- Contains: Bash scripts for format conversion
- Key files:
  - `export-all.sh`: Orchestrator calling all export scripts
  - `export-docx.sh`: Uses pandoc for article → DOCX
  - `export-slides.sh`: Uses marp-cli for slides → PPTX
  - `export-pdf.sh`: Uses pandoc/marp for → PDF

**templates/**
- Purpose: Starting points for new content
- Contains: Markdown templates with YAML front-matter structure
- Key files:
  - `article.md`: Article template (765 bytes)
  - `slides.md`: Marp presentation template (1231 bytes)
  - `research.md`: Research notes template (1100 bytes)

**<topic-name>/ (e.g., epistemic_debt/)**
- Purpose: Complete workspace for a single topic/article
- Contains: All content, research, assets, and exports for one subject
- Key files:
  - `README.md`: Topic status, questions, gaps, next steps
  - `article.md`: Main long-form content
  - `slides.md`: Presentation version of content

**<topic-name>/raw_material/**
- Purpose: Unstructured working notes and brainstorming
- Contains: Conversation logs, early outlines, rough ideas
- Example files:
  - `gsd-convo-1-on-epistemic-debt.md` (90KB conversation log)
  - `outline-v1-epistemic-debt.md` (14KB structured outline)

**<topic-name>/references/**
- Purpose: Literature reviews and source material
- Contains: Citations, academic papers, related research
- Example files: `literature-review-on-epistemic-debt.md`

**<topic-name>/assets/**
- Purpose: Media files for articles and presentations
- Contains: Images, diagrams, charts

**<topic-name>/artifacts/**
- Purpose: Alternative storage for generated artifacts (coexists with exports/)
- Contains: Subdirectories like `articles/`, `presentation/`

**<topic-name>/exports/**
- Purpose: Final publishable outputs
- Contains: DOCX, PPTX, PDF files generated by export scripts

## Naming Conventions

**Files:**
- Lowercase with hyphens: `file-name.md`
- Descriptive names: `outline-v1-epistemic-debt.md` not `notes.md`
- Markdown extension: `.md` for all text content
- Shell scripts: `.sh` extension, executable permissions

**Directories:**
- Lowercase with underscores: `topic_name/`, `raw_material/`
- Standard subdirectories: `raw_material`, `references`, `assets`, `exports`, `artifacts`
- Hidden directories for tooling: `.cursor/`, `.planning/`

**Front-Matter Fields:**
- `title`: String with quotes
- `status`: One of `draft`, `review`, `published`
- `type`: One of `article`, `slides`, `research`
- `audience`: Array of target reader types
- `target_length`: Integer word count
- `created`: Date in `YYYY-MM-DD` format
- `last_updated`: Date in `YYYY-MM-DD` format

## Where to Add New Code

**New Topic:**
1. Create directory: `mkdir new_topic_name`
2. Copy templates: `cp templates/article.md new_topic_name/`
3. Create subdirectories: `mkdir -p new_topic_name/{raw_material,references,assets,exports}`
4. Copy README template: `cp templates/research.md new_topic_name/README.md`

**New Article in Existing Topic:**
- Primary content: `/<topic-name>/article.md` (edit directly)
- Supporting research: `/<topic-name>/raw_material/<descriptive-name>.md`

**New Presentation:**
- Primary slides: `/<topic-name>/slides.md` (edit directly)
- Copy template if missing: `cp templates/slides.md <topic-name>/`

**New Export Format:**
- Script location: `/scripts/export-<format>.sh`
- Follow pattern: Accept topic name as `$1`, validate directory, use external tool, output to `<topic>/exports/`

**New Template:**
- Template location: `/templates/<type>.md`
- Include YAML front-matter structure
- Add placeholder sections with `[GAP: description]` markers

**New Terminology:**
- Add to: `/GLOSSARY.md`
- Group by topic under H2 headings
- Include usage notes for AI assistants

## Special Directories

**.cursor/**
- Purpose: IDE-specific metadata
- Generated: Yes (by Cursor IDE)
- Committed: Yes (contains planning artifacts)

**.planning/**
- Purpose: Codebase documentation
- Generated: Yes (by GSD mapping agents)
- Committed: Yes (part of project documentation)

**<topic>/exports/**
- Purpose: Generated output files
- Generated: Yes (by export scripts)
- Committed: No (should be in .gitignore, though repo not Git-initialized)

**<topic>/artifacts/**
- Purpose: Additional artifact storage
- Generated: Yes (manually or via other tools)
- Committed: Varies (contains presentation/article subdirectories)
